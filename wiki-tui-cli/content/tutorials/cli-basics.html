CLI Basics

Fundamentals of Command Line Interface design

What is a CLI?

A Command Line Interface (CLI) is a text-based interface where users interact with software by typing commands. CLIs are the foundation of Unix/Linux systems and remain essential for developers, system administrators, and power users.

Command Structure

command [options] [arguments]

Example:

git commit -m "Initial commit"
  • git - The command/program
  • commit - The subcommand
  • -m - The option/flag
  • "Initial commit" - The argument

Types of Arguments

Type Example Description
Short flag -v Single letter, can be combined: -la
Long flag --verbose Descriptive name, more readable
Flag with value --output=file.txt Option that requires a value
Positional file.txt Order matters, no prefix

Exit Codes

CLI programs communicate success/failure through exit codes:

0   # Success
1   # General error
2   # Misuse of shell builtins
126 # Command invoked cannot execute
127 # Command not found
128 # Invalid exit argument
130 # Script terminated by Ctrl+C

Streams: stdin, stdout, stderr

  • stdin (0) - Standard input, where the program reads input
  • stdout (1) - Standard output, where the program writes normal output
  • stderr (2) - Standard error, where the program writes error messages
# Redirect stdout to file
command > output.txt

# Redirect stderr to file
command 2> error.txt

# Redirect both to file
command &> all.txt

# Pipe stdout to another command
command1 | command2

Design Principles

📝 One Thing Well

Each command should do one thing and do it well. Use pipes to combine commands.

🔤 Text is Universal

Output should be text-based for easy parsing and piping to other tools.

⚡ Silent by Default

Only output when necessary. Use --verbose for detailed output.

❌ Clear Errors

Error messages should be actionable and explain what went wrong.

Help Documentation

Every CLI should have comprehensive help:

# Standard help flags
myapp --help
myapp -h
myapp help

# Version info
myapp --version
myapp -v
