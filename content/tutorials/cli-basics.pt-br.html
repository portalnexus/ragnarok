Fundamentos de CLI

Fundamentos de design de Command Line Interface

O que é uma CLI?

Uma Command Line Interface (CLI) ou Interface de Linha de Comando é uma interface baseada em texto onde usuários interagem com software digitando comandos. CLIs são a fundação dos sistemas Unix/Linux e permanecem essenciais para desenvolvedores, administradores de sistema e usuários avançados.

Estrutura de Comando

comando [opções] [argumentos]

Exemplo:

git commit -m "Commit inicial"
  • git - O comando/programa
  • commit - O subcomando
  • -m - A opção/flag
  • "Commit inicial" - O argumento

Tipos de Argumentos

Tipo Exemplo Descrição
Flag curta -v Letra única, pode combinar: -la
Flag longa --verbose Nome descritivo, mais legível
Flag com valor --output=arquivo.txt Opção que requer um valor
Posicional arquivo.txt Ordem importa, sem prefixo

Códigos de Saída

Programas CLI comunicam sucesso/falha através de códigos de saída:

0   # Sucesso
1   # Erro geral
2   # Uso incorreto de builtins do shell
126 # Comando não pode ser executado
127 # Comando não encontrado
128 # Argumento de saída inválido
130 # Script terminado com Ctrl+C

Streams: stdin, stdout, stderr

  • stdin (0) - Entrada padrão, onde o programa lê entrada
  • stdout (1) - Saída padrão, onde o programa escreve saída normal
  • stderr (2) - Erro padrão, onde o programa escreve mensagens de erro
# Redirecionar stdout para arquivo
comando > saida.txt

# Redirecionar stderr para arquivo
comando 2> erro.txt

# Redirecionar ambos para arquivo
comando &> tudo.txt

# Encanar stdout para outro comando
comando1 | comando2

Princípios de Design

📝 Uma Coisa Bem

Cada comando deve fazer uma coisa e fazê-la bem. Use pipes para combinar comandos.

🔤 Texto é Universal

Saída deve ser baseada em texto para fácil parsing e encanamento para outras ferramentas.

⚡ Silencioso por Padrão

Apenas faça output quando necessário. Use --verbose para output detalhado.

❌ Erros Claros

Mensagens de erro devem ser acionáveis e explicar o que deu errado.

Documentação de Ajuda

Toda CLI deve ter ajuda abrangente:

# Flags de ajuda padrão
meuapp --help
meuapp -h
meuapp help

# Info de versão
meuapp --version
meuapp -v
