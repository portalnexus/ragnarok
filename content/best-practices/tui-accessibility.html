TUI Accessibility

Making terminal interfaces accessible to all users

Why Accessibility Matters

TUIs are inherently more accessible than GUIs, but there are still important considerations to ensure everyone can use your application effectively.

Screen Reader Support

Best Practices:
  • Use semantic text structure (headings, lists)
  • Provide text alternatives for visual elements
  • Announce state changes (loading, completed)
  • Use clear, descriptive labels

Keyboard Navigation

Ensure all functionality is accessible via keyboard:

KeyAction
TabMove to next interactive element
Shift+TabMove to previous element
EnterActivate selected element
SpaceToggle checkbox/button
↑↓←→Navigate within components
EscClose modal/cancel action
?Show help/keyboard shortcuts

Color Considerations

🎨 Don't Rely on Color Alone

Use icons, text labels, or patterns in addition to color to convey information.

🔆 Sufficient Contrast

Ensure text has at least 4.5:1 contrast ratio against background.

🌈 Color Blind Friendly

Avoid red/green combinations. Use blue/orange or add patterns.

⚫ High Contrast Mode

Support terminal's high contrast settings when available.

Focus Indicators

Always show which element has focus:

# Good: Clear focus indicator
[✓] Option A (focused)
[ ] Option B
[ ] Option C

# Bad: No visible focus
[✓] Option A
[ ] Option B
[ ] Option C

Error Messages

Make errors clear and actionable:

# Good: Clear and actionable
Error: File not found: config.json
Hint: Run 'myapp init' to create a default config

# Bad: Vague
Error: Failed (code: 1)

Progress Indicators

For long operations, provide clear progress:

# Good: Shows progress and ETA
Processing files: [████████░░] 80% (4/5) - ETA: 2s

# Bad: Unclear
Processing...

Text Scaling

Support different terminal font sizes:

  • Use relative sizing where possible
  • Test with various terminal sizes
  • Avoid fixed-width layouts
  • Allow content to wrap naturally

Accessibility Checklist

Before releasing, verify:
  • ☐ All features work with keyboard only
  • ☐ Focus is always visible
  • ☐ Color is not the only indicator
  • ☐ Error messages are clear
  • ☐ Help documentation is available
  • ☐ Works with screen readers
  • ☐ Supports high contrast mode
  • ☐ Text is readable at different sizes
