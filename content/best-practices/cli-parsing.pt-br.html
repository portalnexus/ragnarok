CLI Argument Parsing

Best practices for parsing command line arguments

Overview

Proper argument parsing is crucial for creating user-friendly CLI tools. This guide covers conventions and libraries across different languages.

Convention Standards

GNU Convention:
  • Short options: -f (single letter)
  • Long options: --file (descriptive)
  • Combine short options: -la = -l -a
  • Option with value: -f file.txt or --file=file.txt
  • -- marks end of options

Python - argparse

import argparse

parser = argparse.ArgumentParser(
    description="Process some files."
)
parser.add_argument(
    "files",
    nargs="+",
    help="Files to process"
)
parser.add_argument(
    "-v", "--verbose",
    action="store_true",
    help="Enable verbose output"
)
parser.add_argument(
    "-o", "--output",
    default="output.txt",
    help="Output file (default: output.txt)"
)
parser.add_argument(
    "-f", "--force",
    action="store_true",
    help="Overwrite existing files"
)

args = parser.parse_args()

Python - Click (Modern)

import click

@click.command()
@click.argument("files", nargs=-1)
@click.option("-v", "--verbose", is_flag=True)
@click.option("-o", "--output", default="output.txt")
@click.option("-f", "--force", is_flag=True)
def process(files, verbose, output, force):
    """Process some files."""
    if verbose:
        click.echo(f"Processing {files}")
    click.echo(f"Output: {output}")

if __name__ == "__main__":
    process()

Rust - clap

use clap::{Parser, Subcommand};

#[derive(Parser)]
#[command(name = "myapp")]
#[command(about = "Process some files", long_about = None)]
struct Args {
    /// Files to process
    #[arg(required = true)]
    files: Vec,

    /// Enable verbose output
    #[arg(short, long)]
    verbose: bool,

    /// Output file
    #[arg(short, long, default_value = "output.txt")]
    output: String,

    /// Overwrite existing files
    #[arg(short, long)]
    force: bool,
}

fn main() {
    let args = Args::parse();
}

Go - flag

package main

import (
    "flag"
    "fmt"
)

func main() {
    verbose := flag.Bool("v", false, "verbose output")
    output := flag.String("o", "output.txt", "output file")
    force := flag.Bool("force", false, "overwrite files")
    
    flag.Parse()
    files := flag.Args()
    
    fmt.Printf("Files: %v\n", files)
    fmt.Printf("Verbose: %v\n", *verbose)
    fmt.Printf("Output: %s\n", *output)
}

Node.js - Commander

import { Command } from 'commander';
const program = new Command();

program
    .name('myapp')
    .description('Process some files')
    .argument('<files...>', 'files to process')
    .option('-v, --verbose', 'verbose output')
    .option('-o, --output <file>', 'output file', 'output.txt')
    .option('-f, --force', 'overwrite existing files')
    .action((files, options) => {
        console.log('Files:', files);
        console.log('Verbose:', options.verbose);
        console.log('Output:', options.output);
    });

program.parse();

Best Practices

📝 Descriptive Help

Every option should have clear help text. Users shouldn't need external docs.

⚡ Sensible Defaults

Provide reasonable defaults so users don't need to specify every option.

❌ Clear Errors

When parsing fails, explain what went wrong and how to fix it.

🔄 Backward Compatible

Don't break existing scripts. Deprecate options gradually.
