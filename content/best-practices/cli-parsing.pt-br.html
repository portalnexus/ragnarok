Análise de Argumentos CLI

Melhores práticas para analisar argumentos de linha de comando

Visão Geral

A análise adequada de argumentos é crucial para criar ferramentas CLI amigáveis. Este guia cobre convenções e bibliotecas em diferentes linguagens de programação.

Padrões de Convenção

Convenção GNU:
  • Opções curtas: -f (letra única)
  • Opções longas: --file (descritivo)
  • Combinar opções curtas: -la = -l -a
  • Opção com valor: -f arquivo.txt ou --file=arquivo.txt
  • -- marca o fim das opções

Python - argparse

A biblioteca padrão para análise de argumentos em Python:

import argparse

parser = argparse.ArgumentParser(
    description="Processar alguns arquivos."
)
parser.add_argument(
    "files",
    nargs="+",
    help="Arquivos para processar"
)
parser.add_argument(
    "-v", "--verbose",
    action="store_true",
    help="Habilitar saída detalhada"
)
parser.add_argument(
    "-o", "--output",
    default="output.txt",
    help="Arquivo de saída (padrão: output.txt)"
)
parser.add_argument(
    "-f", "--force",
    action="store_true",
    help="Sobrescrever arquivos existentes"
)

args = parser.parse_args()

Python - Click (Moderno)

Click é uma biblioteca moderna e elegante para criação de CLIs:

import click

@click.command()
@click.argument("files", nargs=-1)
@click.option("-v", "--verbose", is_flag=True)
@click.option("-o", "--output", default="output.txt")
@click.option("-f", "--force", is_flag=True)
def process(files, verbose, output, force):
    """Processar alguns arquivos."""
    if verbose:
        click.echo(f"Processando {files}")
    click.echo(f"Saída: {output}")

if __name__ == "__main__":
    process()

Rust - clap

Clap é a biblioteca mais popular para parsing de argumentos em Rust:

use clap::{Parser, Subcommand};

#[derive(Parser)]
#[command(name = "myapp")]
#[command(about = "Processar alguns arquivos", long_about = None)]
struct Args {
    /// Arquivos para processar
    #[arg(required = true)]
    files: Vec<String>,

    /// Habilitar saída detalhada
    #[arg(short, long)]
    verbose: bool,

    /// Arquivo de saída
    #[arg(short, long, default_value = "output.txt")]
    output: String,

    /// Sobrescrever arquivos existentes
    #[arg(short, long)]
    force: bool,
}

fn main() {
    let args = Args::parse();
}

Go - flag

A biblioteca padrão flag do Go para parsing de argumentos:

package main

import (
    "flag"
    "fmt"
)

func main() {
    verbose := flag.Bool("v", false, "saída detalhada")
    output := flag.String("o", "output.txt", "arquivo de saída")
    force := flag.Bool("force", false, "sobrescrever arquivos")

    flag.Parse()
    files := flag.Args()

    fmt.Printf("Arquivos: %v\n", files)
    fmt.Printf("Detalhado: %v\n", *verbose)
    fmt.Printf("Saída: %s\n", *output)
}

Node.js - Commander

Commander é a biblioteca mais popular para CLIs em Node.js:

import { Command } from 'commander';
const program = new Command();

program
    .name('myapp')
    .description('Processar alguns arquivos')
    .argument('<files...>', 'arquivos para processar')
    .option('-v, --verbose', 'saída detalhada')
    .option('-o, --output <arquivo>', 'arquivo de saída', 'output.txt')
    .option('-f, --force', 'sobrescrever arquivos existentes')
    .action((files, options) => {
        console.log('Arquivos:', files);
        console.log('Detalhado:', options.verbose);
        console.log('Saída:', options.output);
    });

program.parse();

Melhores Práticas

📝 Ajuda Descritiva

Cada opção deve ter texto de ajuda claro. Usuários não deveriam precisar de documentação externa.

⚡ Padrões Sensatos

Forneça padrões razoáveis para que usuários não precisem especificar cada opção.

❌ Erros Claros

Quando a análise falhar, explique o que deu errado e como corrigir.

🔄 Retrocompatível

Não quebre scripts existentes. Descontinue opções gradualmente.
